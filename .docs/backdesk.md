# BackDesk サービス・システム ドキュメント

BackDesk のサービス概要と、それを支える業務管理システムの内容をまとめたドキュメントです。  
提案書に基づいて整理しています。

## 2. システム概要（BackDesk 基盤）

BackDesk サービスを支える業務管理システム。稼働・請求・支払を統合的に管理し、業務効率化・情報の正確性を実現する。

---

## 3. 業務フロー

### 3.1 アサイン（アサイン前〜アサイン登録まで）

**アサイン前：OurDesk と秘書の準委任契約締結時**

1. **全権ユーザ（PM秘書）** と **秘書** が **ヒアリング** を行う。
2. ヒアリングを経て **契約** を締結する。
3. 契約締結後、**全権ユーザ（PM秘書）** が BackDesk システムへ **秘書登録** を行う。
   - 登録内容：契約締結した秘書の個人情報、口座情報、秘書ランクなど。
   - 秘書ランクは「初級」「中級」「上級」とし、ヒアリング内容やスキルに応じて決定する。

**仕事依頼からアサイン登録まで**

1. **顧客** が OurDesk に仕事を依頼する（**仕事依頼**）。
2. **全権ユーザ（PM秘書）** が依頼内容に適した秘書を選び、顧客へのアサインを決める。
3. **全権ユーザ（PM秘書）** と **秘書** が **ヒアリング** を行う。
4. **秘書** が仕事内容・時給を確認し、受けるかどうか判断する。
5. アサインが決まったら、**全権ユーザ（PM秘書）** が BackDesk システムへ **アサイン登録** を行う。
   - 登録内容：会社名、秘書名、業務ランクなど。
6. システムで「どの顧客にどの秘書がアサインしているか」を閲覧・管理する。

### 3.2 業務実績の管理

**業務の登録・確認・承認・差戻し**

1. **秘書** が **業務登録** を行う。
   - 備考入力欄あり。稼働内容を随時記載可能。
   - 同じアサイン先の秘書も閲覧可能。
2. BackDesk システムで内容確認のうえ、**全権ユーザ（PM秘書）** が **承認** または **差戻し** を行う。
   - 差戻す場合は理由を記載可能。
   - 一括での承認・差戻しも可能。
   - 秘書の稼働が適切か精査し、承認・差戻しを実施する。
3. **顧客** は秘書の **稼働状況** を随時閲覧可能。不適切な稼働内容に対しては **差戻し申請** をシステム経由で行える。

**差戻し後の流れ**

1. **秘書** が差戻しされた業務の内容を確認し、**修正・再登録** を行う。
2. BackDesk システムで内容確認のうえ、**全権ユーザ（PM秘書）** が再登録された業務について承認・差戻しを行う。

### 3.3 請求処理

**顧客請求書の提出**

1. **全権ユーザ（OurDesk 事務）** が BackDesk システムで確認を行う。
2. システムから **顧客請求データ（CSV形式）** を **CSV 出力** する。
3. 出力した CSV を **MFクラウド** にインポートする。
4. MFクラウド上で **顧客請求書** を作成し、**顧客** に請求書を送付する。
5. 送付後、**請求ステータス** を「請求済」に更新する。

**秘書請求書の提出**

1. **秘書** が BackDesk システムで確認を行う。
2. システムから **秘書請求書** を **Excel 出力** する（登録済みの口座情報を反映）。
3. **秘書** が Excel 出力内容を確認する。
4. **全権ユーザ（OurDesk 事務）** がシステム上でデータを確認する。
5. 秘書請求書を PDF 化して送付する。

**共通**  
稼働内容に不備があった場合は、業務実績を修正したうえで再発行する。

### 3.4 【月末】支払処理

**顧客から OurDesk への送金**

1. **顧客** が顧客請求書に基づき、OurDesk へ業務委託料を送金する。請求ステータスで状況を確認する。
2. **全権ユーザ（OurDesk 事務）** が銀行口座で入金を確認し、BackDesk システム上で **請求ステータス** を「入金済」に変更する。

**OurDesk から秘書への振込**

1. **全権ユーザ（OurDesk 事務）** が秘書請求書に基づき、秘書へ業務委託料を送金する。BackDesk システム上で **支払ステータス** を「振込済」に変更する。
2. **秘書** が銀行口座および支払ステータスで状況を確認する。

## 4. 用語定義

| 用語 | 説明 |
|------|------|
| **ヒアリング** | 全権ユーザが、顧客や秘書に対して要望や現状を聞き取ること。 |
| **秘書登録** | 全権ユーザが、契約を結んだ秘書をシステムに登録すること。 |
| **秘書ランク** | ヒアリング内容やスキルに応じて決まる秘書のランク。「初級」「中級」「上級」がある。 |
| **業務ランク** | 業務ごとの難易度に応じて設定される業務のランク。 |
| **BackDesk システム** | 本ドキュメントでいう「BackDesk の業務管理システム」を指す。 |
| **アサイン登録** | 全権ユーザが、アサイン内容をシステムに登録すること。 |
| **アサイン** | 1顧客に秘書を割り当てるまでの流れ、および割当済みの状態。月次単位で管理。 |
| **業務登録** | 秘書が稼働内容を業務実績としてシステムに入力・登録すること。 |
| **稼働状況** | 秘書の稼働時間や業務内容。 |
| **承認** | 全権ユーザが、秘書の業務実績に不備がない場合に承認すること。 |
| **差戻し** | 全権ユーザが、業務実績に不備や不整合がある場合に差戻しすること。 |
| **差戻し申請** | 顧客が、業務実績に不備等がある場合に全権ユーザへ差戻しを申請すること。 |
| **再登録** | 秘書が業務実績を再登録すること。 |
| **MFクラウド** | マネーフォワードクラウド（外部）。顧客請求書発行に利用。システムとは CSV のやりとりのみ。 |
| **顧客請求書** | OurDesk が MFクラウドで発行する月次請求書（1顧客に対し、複数秘書分の業務委託料を請求）。 |
| **秘書請求書** | 秘書がシステムから Excel 出力し、PDF 化して OurDesk に提出する月次請求書。 |
| **請求ステータス** | 顧客への請求状況および顧客からの入金状況。 |
| **支払ステータス** | 秘書への支払い状況。 |
| **単価調整** | 一定期間継続したアサインで発生する業務委託料の上方調整。 |

| アクター | 想定人数 | 説明 | システム上で行う主な業務 |
|----------|----------|------|---------------------------|
| **全権ユーザ** | 約5名 | OurDesk 社員。アカウント発行・ほぼ全機能にアクセス可能。 | パスワード変更、各利用者アカウントの作成・変更・削除・詳細閲覧、業務ランク・秘書ランクの登録・変更・削除、アサインの登録・削除・一覧、業務の承認・差戻し・一覧、秘書請求・顧客請求の一覧・詳細・ステータス変更、売上・支払コストの閲覧 |
| **顧客** | 約40名 | 準委任契約を結んだ依頼者。1社に複数担当者（同一情報にアクセス）。 | パスワード変更、アサイン一覧、秘書の業務時間・内容の閲覧、請求詳細・請求ステータスの閲覧 |

| **MFクラウド** | — | 外部システム。顧客請求書発行に使用。 | システムから出力した CSV をインポートして請求書を発行 |

## 6. 機能要求（必須機能の整理）

### 6.1 共通・認証・マイページ

- ログイン・ログアウト（メールアドレス＋パスワード）
- ユーザー種別ごとにログイン URL を分離
- パスワードリセット（ログイン前）
- 権限：秘書・顧客・全権ユーザの3区分
- マイページ：名前・メールアドレスの確認、ログイン中にパスワード変更

### 6.2 マスタ管理

- 業務ランク：登録・一覧・詳細・変更・削除
- 秘書ランク：登録・一覧・詳細・変更・削除

### 6.3 アカウント管理

- **全権ユーザ**：登録・一覧・詳細・変更・削除（自身の削除は不可）
- **秘書**：全権が登録・一覧・詳細・変更・削除。登録項目は名前・住所・メール・電話・緊急連絡先・続柄。秘書は名前・住所・メール・電話は変更不可（緊急連絡先は変更可）。削除は論理削除。一般秘書⇔PM秘書の権限変更が可能。
- **顧客**：全権が顧客・担当者の登録・一覧・詳細・変更・削除。顧客一覧は PM秘書・全権が閲覧。1顧客に複数担当者。顧客・担当者はメール・電話は変更不可（担当者の所属部署は変更可）。顧客詳細から当該顧客の秘書稼働・請求履歴を閲覧可能。

### 6.4 月次アサイン・業務実績

- **アサイン**：月次単位で登録・一覧・削除（変更は不可）。一括複製で新規登録可能。1顧客に複数秘書。細かい単価調整（例：100円UP）が可能。何か月目か・稼働ステータスの確認が可能。
- **業務実績**：秘書が業務の登録・一覧・変更・削除。全権が承認業務（承認・差戻し、一括承認・一括差戻し、差戻し理由入力、承認キャンセル）。秘書は自身の月別業務量、顧客は担当秘書の業務量を閲覧。

### 6.5 請求管理

- 秘書の稼働登録に連動して顧客請求金額・秘書への支払コストを自動計算・表示。
- 全権：秘書請求・顧客請求の一覧・詳細、請求ステータス・支払ステータスの変更・閲覧、備考の入力・閲覧。
- 秘書：請求詳細の閲覧、Excel 出力、支払ステータス・備考の閲覧。請求書に口座情報を表示。
- 顧客：請求詳細・請求ステータス・備考の閲覧。
- 全権：顧客請求データを CSV 出力。MFクラウドにインポートして顧客請求書を発行。

### 6.6 売上管理

- 顧客別・月別の売上の可視化
- 月別の秘書への支払コストの可視化
- 売上比較の図の出力

## 7. 技術・非機能（提案書ベース）

- **技術スタック**：Java（Servlet/JSP）、JavaScript（jQuery）、PostgreSQL、Nginx、Apache Tomcat10、AlmaLinux10、さくらVPS。MFクラウドは API 連携ではなく CSV 出力→インポート。
- **規模想定**：全権約100名、顧客最大1000社・利用者最大5000名、秘書最大1000名。応答は3秒を目安。
- **運用**：24時間稼働を原則（23時〜6時の停止は影響が小さい範囲で許容）。バックアップ自動化、定期メンテナンス、SLA（稼働率99.9%目標など）を設定。
