## 0. このサイトのデザイン構造（最重要）

このサイトは **2つの異なる美学が意図的に共存している**。

### レイヤー 1 ── ナビゲーション（StaggeredMenu）
洗練・エディトリアル・モノクローム。  
大文字タイポグラフィ、ネガティブ letter-spacing、黒と白のみ。  
ユーザーがサイトに入る「入口」として都会的な緊張感を持つ。

### レイヤー 2 ── ページコンテンツ
有機的・人間的・温かい。  
クリーム背景、手書きライン、Blob 装飾。  
OurDesk の「伴走型」「寄り添い」という姿勢を体現する。

### なぜこの対比が成立するか
骨格（ナビ）がクールだからこそ、中身（コンテンツ）の温かさが映える。  
どちらかに統一しようとしない。対比そのものがデザインの意図。

ただしコンテンツ側が温かみを出しすぎると（Blob を全セクションに入れる・全体をクリームにする）
ナビとのギャップが大きくなりすぎる。  
コンテンツ側は「引き算」も意識する。Blob を置かないセクション、白いセクションを意図的に挟む。

### このサイトを見る2種類のユーザー

このサイトには **閲覧者が2種類いる**。設計判断に迷ったときは常にこの両者を意識する。

| ユーザー | 求めているもの |
|---------|--------------|
| **求職者（働きたい女性）** | 自分の想いや可能性をちゃんと見てくれる場所かどうか。安心感・共感・対等な関係性。 |
| **クライアント企業（秘書・事務の発注側）** | 人を丁寧に育てて送り出す仕組みがある会社かどうか。信頼感・誠実さ・任せられる安心感。 |

この2つは実は**同じデザイン言語で成立する**。「育む・丁寧・誠実」が共通軸。

## 1. 会社・ブランドの理解

### OurDesk株式会社とは

**業種**: バックオフィスサポート・BPO  

**何をしている会社か**:  
「働きたい」気持ちを持つ女性に対して、スキル研修・キャリアコンサルタントによる面談・仕事のアサインまでを一貫して行う伴走型キャリア支援会社。  
秘書業務・事務業務などのバックオフィス案件に人材をつなぐことが主事業。

**ターゲット**:  
子育て中・未経験・ライフイベントで働き方が変わった女性が主。  
月数時間からフルタイムまで、ライフスタイルに合わせた柔軟な働き方を提供する。

**最も重要なメッセージ**:  
スキルや経歴より「**誰かの役に立ちたい**」「**支える仕事がしたい**」という想いを重視する。  
未経験でも、今の可能性より**これからの成長**を共に描く会社。  
→ 「効率」ではなく「想い」と「育み」が起点の会社。

### 目指さないトーン
- 過剰な有機デザイン
- 派手すぎるアニメーション
- キラキラしたラグジュアリー感
- **ふわふわした・かわいい印象**（求職者を軽く見ているような印象につながる）
- 「完成されたプロ集団」感（未経験OKの会社なので実態と乖離する）

## 2. カラールール

### ブランドアクセントカラー

```
黄: #FDD000
オレンジ: #F08300
```

この2色がブランドカラー。グラデーション・単色、どちらの形でも使う。

### グラデーションの使い方

**基本的にどこに使ってもよい**（ボタン全面・カード背景・セクション背景を含む）。

ただし以下は守ること:

- 色がゆっくり・なだらかに黄→オレンジに変わるグラデーションを使う
- **光沢感・輝き感が出るような強いグラデーションは使わない** ❌  
  → ギラついた・派手すぎる表現はブランドのトーンと合わない
- 同じページで多用しすぎると単調になるので、密度を意識する

### 背景色パレット

| 用途 | カラー |
|------|--------|
| 暖かいクリーム | `#fffdf5` |
| やや白に近いクリーム | `#fefcf7` |
| ホワイトセクション | `#ffffff` |
| ダーク背景（CTA など） | `gray-900` / `gray-800` |

ページは白・クリームの両方を使い、**白セクションを意図的に挟む**。全セクションをクリームにしない。

### テキストカラー

| 用途 | 値 |
|------|----|
| ボディテキスト（body） | `#333333` |
| 見出し | `text-gray-800` / `text-gray-900` |
| 本文 | `text-gray-600` / `text-gray-700` |
| サブテキスト | `text-gray-500` |
| アクセントテキスト | `text-orange-600` |

### カラーの自由度
青・緑を使うことは問題ない。  
ただしメインのブランドカラー（黄→オレンジ）と色相が離れた色を多用すると統一感が崩れるので、
アクセントや情報色（エラー、バッジ等）に限定して使う。

## 3. タイポグラフィ

### フォントの使い分け

| 用途 | フォント |
|------|----------|
| 装飾的な大きな数字（01, 02, 03） | Crimson Pro（`.serif` クラス） |
| ヒーロー和文見出し（感情的なコピー） | Shippori Mincho B1 |
| 通常見出し・本文 | システムフォント（Tailwind デフォルト） |

> StaggeredMenu は uppercase + negative letter-spacing の英字タイポグラフィを使う。
> ページコンテンツ側では uppercase を多用しない（ナビとのコントラストを守る）。

### テキスト修飾ルール（必ず適用）
- **見出し: `text-balance`**
- **本文: `text-pretty`**
- `leading-relaxed` を本文に使う

### フォントサイズ階層
```

セクション見出し: text-2xl md:text-4xl
カード見出し   : text-xl md:text-2xl

サブテキスト   : text-sm
注釈          : text-xs md:text-sm
```

## 4. 手書き風ライン（HandwrittenLine コンポーネント）

### 概要

見出し下・セクション区切りのラインは **HandwrittenLine のみ**使う。`.divider-line` は使用禁止。  
`/components/ui/HandwrittenLine.tsx` を使う。

SVG ファイルは `/public/lines/line1.svg` 〜 `/public/lines/line6.svg` に置かれている。  
色を CSS で制御したい場合はインライン SVG で読み込む（`img` タグでは `fill` 変更不可）。

```tsx
<HandwrittenLine variant={1} color="#FDD000" width={100} />
<HandwrittenLine variant={3} color="rgba(240,131,0,0.6)" width={80} align="left" />
```

### バリアントの特徴

| variant | 特徴 | 存在感 |
|---------|------|--------|
| 1 | 横に長い細波。自然な揺れ | ふつう |
| 2 | 短め・波打ち | やや小 |
| 3 | 超フラット・繊細 | 小 |
| 4 | 太い筆タッチ | 強 |
| 5 | 細かく震えた繊細なライン | 小 |
| 6 | 複雑・有機的な太筆 | 最強 |

### 不規則性のルール
- **同一ページで同じ variant を連続して使わない**
- **色も単調に繰り返さない**
- variant 4・6 は存在感が強いのでアクセントとして使う（全セクションに使わない）
- **ラインを使わないセクションを意図的に作る**

#### Recruit ページ割り当て例

| セクション | variant | color |
|-----------|---------|-------|
| IntroSection | 1 | `#FDD000` |
| WorkStylesSection | 3 | `#F08300` |
| FeaturesSection | 2 | `rgba(240,131,0,0.6)` |
| CareerPathsSection | 5 | `#FDD000` |
| FlowSection | 4 | `rgba(178,186,230,0.8)` |
| FaqSection | 6 | `#F08300` |

## 5. Blob 装飾（BlobDecoration コンポーネント）

### 基本思想
**画面内に収めない。はみ出す。重なる。隠れる。**

Blob が表現するのは**生き生きとした成長感・人間的な温かさ**であり、装飾的なかわいさではない。

### 不規則性のルール
- **同一ページで同じ shape を連続使用しない**
- **同一ページで同じ color を隣接セクションで繰り返さない**
- 配置は「右上・左下・右下・左上」をローテーション

### Blob カラーパレット

| カラー | 用途 |
|--------|------|
| `rgba(253, 232, 166, 0.28)` | クリーム・イエロー |
| `rgba(235, 180, 178, 0.2)` | ピンク |
| `rgba(178, 186, 230, 0.2)` | ラベンダー |
| `rgba(249, 115, 22, 0.25)` | オレンジ（CTA・強調） |

透明度は 0.15〜0.28 の範囲に保つ。

## 6. 画像・セクションの切り取り方

**画像は大胆にトリミングする。**

セクション内の画像は「収まりよく」置くのではなく、画面やコンテナに対して大胆に切り出す。  
具体例:
- `hero-photo` のように角丸を片側だけ大きくする（`border-radius: 0 0 0 12rem`）
- 画像がセクション境界をはみ出す構成
- アスペクト比を縦長・横長どちらにも躊躇なく使う
- `object-cover` で被写体が画面端で切れることを恐れない

Wave Divider のカラーは固定しない。セクションの背景色に合わせて自由に決める。

## 8. セクション区切り・レイヤリング

### セクション間の背景切り替えは直線にしない

背景色が変わるセクション境界を **直線でぶつける** のは避ける。  
波型・曲線・有機的なシェイプなど、何らかの方法で滑らかにつなぐことを推奨。

Recruit ページでは `WaveDivider` コンポーネントを使っている。参考実装として：

```tsx
{/* 白 → クリームの境界 */}
<WaveDivider bgColor="#ffffff" fillColor="#fffdf5" className="z-[6]" />

{/* クリーム → 白の境界（flip で波の向きを反転） */}
<WaveDivider bgColor="#fffdf5" fillColor="#ffffff" flip className="z-[10]" />
```

`bgColor` = 手前のセクションの背景色  
`fillColor` = 次のセクションの背景色  
`flip` = 波の向きを上下反転したいとき

境界の表現方法は WaveDivider に限らない。セクションの雰囲気に合わせて選ぶ。

Wave/区切りの色はセクションの背景に合わせて自由に決める。固定しない。

Recruit ページのレイヤー構造:
```
RecruitWorkStylesSection  z-[5]
RecruitCareerPathsSection z-[7]
RecruitFlowSection        z-[9]
RecruitFaqSection         z-[11]
RecruitCtaSection         z-[14]
```
新規セクションを既存の間に挟む場合は奇数の z-index を使う。

## 10. StaggeredMenu との共存ルール

StaggeredMenu は独自の美学（uppercase・モノクローム・大字タイポ）を持つ。
ページコンテンツはこれと直接競合しない。

### やること
- ページコンテンツは独自の「温かさ」を保つ
- 白セクションを意図的に挟んでナビとのつなぎ目を緩衝する

### やらないこと
- ページコンテンツ側で uppercase を多用する（ナビとぶつかる）
- ページコンテンツ全体を有機的にしすぎてナビとの乖離を大きくする

## 10.5 SKILL.md との関係

プロジェクトには汎用 UI 規約として `SKILL.md` がある。  
このドキュメントと衝突する箇所は、**本ドキュメントを優先する**。

| 項目 | SKILL.md | 本ドキュメントの扱い |
|------|----------|---------------------|
| グラデーション | 明示的リクエストなしは禁止 | ブランドカラー表現として意図的に使用。本ドキュメント優先 |
| letter-spacing | 明示的リクエストなしは変更禁止 | StaggeredMenu の美学として意図的に使用。本ドキュメント優先 |
| アニメーション | 明示的リクエストなしは禁止 | Blob の drift アニメーションは設計に組み込まれている。本ドキュメント優先 |
| z-index 管理 | 固定スケールを使うこと | **SKILL.md に従う**（実装都合なので統一した方がよい） |

## 11. NGリスト

- **インラインのグラデーション棒（`span` で `w-[80px] h-[4px] rounded-full` の黄〜オレンジグラデーション）は永久に使用禁止。** ❌
- **`.divider-line`（グラデーション直線）は使用禁止。** 見出し下・セクション区切りのラインは **HandwrittenLine のみ**使う。 ❌
- 光沢感・輝き感のあるギラついたグラデーションを使う ❌
- 見出しに `text-balance` を付けない ❌
- 本文に `text-pretty` を付けない ❌
- ページコンテンツで uppercase を多用する ❌
- prefers-reduced-motion 対応を書き忘れる ❌
- 同一ページで Blob を同じ色・同じ配置で連続する ❌
- 画像をセクション内に「きれいに収める」だけにする ❌
- **ふわふわ・かわいい印象のデザインにする（求職者を軽く見た表現になる）** ❌
- **cheecolo の「嗜好品感・小さなごほうび」的トーンを参考にする（造形語彙のみ参照）** ❌
- **クライアント企業視点（任せられるか・信頼できるか）を忘れる** ❌
- **セクション間の背景色を直線でぶつける（波型・曲線など滑らかな境界を推奨）** ❌
