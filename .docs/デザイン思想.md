# デザイン思想ドキュメント — OurDesk コーポレートサイト

> Cursor がコードを生成・修正するときの設計思想の基準。迷ったときはここに戻る。
>
> **【重要】** Recruit ページが全ページのデザイン基準。新規ページ作成時は Recruit の実装を参照しながらこのドキュメントに従うこと。ただし Recruit 固有のセクション構成・データをそのままコピーしてはいけない。各ページに合った構成を設計すること。

## 0. このサイトのデザイン構造（最重要）

このサイトは **2つの異なる美学が意図的に共存している**。

### レイヤー 1 ── ナビゲーション（StaggeredMenu）
洗練・エディトリアル・モノクローム。  
大文字タイポグラフィ、ネガティブ letter-spacing、黒と白のみ。  
ユーザーがサイトに入る「入口」として都会的な緊張感を持つ。

### レイヤー 2 ── ページコンテンツ
有機的・人間的・温かい。  
クリーム背景、手書きライン、Blob 装飾。  
OurDesk の「伴走型」「寄り添い」という姿勢を体現する。

### なぜこの対比が成立するか
骨格（ナビ）がクールだからこそ、中身（コンテンツ）の温かさが映える。  
どちらかに統一しようとしない。対比そのものがデザインの意図。

ただしコンテンツ側が温かみを出しすぎると（Blob を全セクションに入れる・全体をクリームにする）ナビとのギャップが大きくなりすぎる。コンテンツ側は「引き算」も意識する。Blob を置かないセクション、白いセクションを意図的に挟む。

### このサイトを見る2種類のユーザー

| ユーザー | 求めているもの |
|---------|--------------|
| **求職者（働きたい女性）** | 自分の想いや可能性をちゃんと見てくれる場所かどうか。安心感・共感・対等な関係性。 |
| **クライアント企業（秘書・事務の発注側）** | 人を丁寧に育てて送り出す仕組みがある会社かどうか。信頼感・誠実さ・任せられる安心感。 |

この2つは実は**同じデザイン言語で成立する**。「育む・丁寧・誠実」が共通軸。

## 1. 会社・ブランドの理解

### OurDesk株式会社とは

**業種**: バックオフィスサポート・BPO  

**何をしている会社か**:  
「働きたい」気持ちを持つ女性に対して、スキル研修・キャリアコンサルタントによる面談・仕事のアサインまでを一貫して行う伴走型キャリア支援会社。秘書業務・事務業務などのバックオフィス案件に人材をつなぐことが主事業。

**ターゲット**:  
子育て中・未経験・ライフイベントで働き方が変わった女性が主。月数時間からフルタイムまで、ライフスタイルに合わせた柔軟な働き方を提供する。

**最も重要なメッセージ**:  
スキルや経歴より「**誰かの役に立ちたい**」「**支える仕事がしたい**」という想いを重視する。未経験でも、今の可能性より**これからの成長**を共に描く会社。「効率」ではなく「想い」と「育み」が起点の会社。

### 目指さないトーン
- 過剰な有機デザイン
- 派手すぎるアニメーション
- キラキラしたラグジュアリー感
- **ふわふわした・かわいい印象**（求職者を軽く見ているような印象につながる）
- 「完成されたプロ集団」感（未経験OKの会社なので実態と乖離する）

## 2. 新規ページの骨格テンプレート

新規ページを作るときはこの骨格を起点にする。セクション数・順序はページの目的に応じて調整してよいが、背景色のリズム・HeroSection の扱い・CTAの配置は守ること。

```tsx
export default function SomePage() {
    return (
        <>
            <BreadcrumbJsonLdServer path="/some-path/" name="ページ名" />
            <main className="min-h-screen bg-white relative">

                {/* ① HeroSection + 直下セクション: 同じクリーム背景でラップして一体化 */}
                <div className="relative z-[1] bg-[#fffdf5]">
                    {/* Blob装飾（任意: ヒーロー周辺を彩る） */}
                    <BlobDecoration ... />
                    <HeroSection title="..." description="..." activeIndex={N} />
                    <SomeIntroSection ... />       {/* ← 白背景と一体化させたくない最初のセクション */}
                </div>

                {/* ② Wave: クリーム→白 */}
                <WaveDivider bgColor="#fffdf5" fillColor="#ffffff" className="z-[2]" />

                {/* ③ メインコンテンツ（白・クリームを交互に。同色が続く境界にWaveDividerは置かない） */}
                <SomeMainSection />               {/* bg-white */}

                <WaveDivider bgColor="#ffffff" fillColor="#fffdf5" className="z-[4]" />

                <SomeSubSection />                {/* bg-[#fffdf5] */}

                <WaveDivider bgColor="#fffdf5" fillColor="#ffffff" className="z-[6]" />

                {/* ④ CTAセクション（ArchClip使用 or ダーク背景） */}
                {/* ArchClip使用時: section の bg-* は前セクションの背景色に合わせる */}
                {/* ArchClip使用時: 直前にWaveDividerを置かない */}
                <SomeCtaSection />

            </main>
        </>
    )
}
```

### z-index の割り当て方針

ページ全体で一貫したスケールを使う。セクションごとに奇数を飛ばして割り当て、間に挟む場合は空き番号を使う。

| セクション位置 | z-index 目安 |
|--------------|-------------|
| HeroSection ラッパー | 1 |
| WaveDivider 1枚目 | 2 |
| コンテンツ順に +2 ずつ | 3, 5, 7... |
| WaveDivider は直後のセクション +1 | 4, 6, 8... |
| CTAセクション | 最大値 |

## 3. カラールール

### ブランドアクセントカラー

```
黄: #FDD000
オレンジ: #F08300
```

この2色がブランドカラー。グラデーション・単色、どちらの形でも使う。

### グラデーションの使い方

基本的にどこに使ってもよい（ボタン全面・カード背景・セクション背景を含む）。ただし以下は守ること:

- 色がゆっくり・なだらかに黄→オレンジに変わるグラデーションを使う
- **光沢感・輝き感が出るような強いグラデーションは使わない** ❌

### 背景色パレット

| 用途 | カラー |
|------|--------|
| 暖かいクリーム | `#fffdf5` |
| やや白に近いクリーム | `#fefcf7` |
| ホワイトセクション | `#ffffff` |
| ダーク背景（CTA など） | `gray-900` / `gray-800` |

ページは白・クリームの両方を使い、**白セクションを意図的に挟む**。全セクションをクリームにしない。

### テキストカラー

| 用途 | 値 |
|------|----|
| ボディテキスト（body） | `#333333` |
| 見出し | `text-gray-800` / `text-gray-900` |
| 本文 | `text-gray-600` / `text-gray-700` |
| サブテキスト | `text-gray-500` |
| アクセントテキスト | `text-orange-600` |

青・緑を使うことは問題ない。ただしメインのブランドカラーと色相が離れた色を多用すると統一感が崩れるので、アクセントや情報色（エラー、バッジ等）に限定する。

| 用途 | フォント |

## 5. 手書き風ライン（HandwrittenLine コンポーネント）

### 概要

見出し下・セクション区切りのラインは **HandwrittenLine のみ**使う。`.divider-line` は使用禁止。  
`/components/ui/HandwrittenLine.tsx` を使う。

```tsx
<HandwrittenLine variant={1} color="#FDD000" width={100} />
<HandwrittenLine variant={3} color="rgba(240,131,0,0.6)" width={80} align="left" />
```

### バリアントの特徴

| variant | 特徴 | 存在感 |
|---------|------|--------|
| 1 | 横に長い細波。自然な揺れ | ふつう |
| 2 | 短め・波打ち | やや小 |
| 3 | 超フラット・繊細 | 小 |
| 4 | 太い筆タッチ | 強 |
| 5 | 細かく震えた繊細なライン | 小 |
| 6 | 複雑・有機的な太筆 | 最強 |

### 不規則性のルール
- **同一ページで同じ variant を連続して使わない**
- **色も単調に繰り返さない**
- variant 4・6 は存在感が強いのでアクセントとして使う（全セクションに使わない）
- **ラインを使わないセクションを意図的に作る**

### HeroSection 直下のラインには `delayWhenInViewMs` を必ず指定する（厳守）

HeroSection 直下（ファーストビュー内）の HandwrittenLine はページロード時点で既にviewport内にあるため、マウント直後に即発火して HeroSection のアニメーションと重なる。これを防ぐため必ず `delayWhenInViewMs={2000}` を指定すること。

```tsx
{/* HeroSection 直下: delayWhenInViewMs 必須 */}
<HandwrittenLine variant={1} color="#FDD000" width={140} align="left" delayWhenInViewMs={2000} />

{/* スクロール後のセクション: 不要（viewport 外から入ってくるため） */}
<HandwrittenLine variant={2} color="#F08300" width={80} />
```

**アニメーションのタイミング制御まとめ**

| 項目 | 場所 | 値 |
|------|------|----|
| 発火の遅延 | 各セクションの `delayWhenInViewMs` prop | ファーストビュー内は `2000`（ms） |
| アニメの長さ | `globals.css` `.handwrite-line.is-visible` | `0.9s` |
| イージング | 同上 | `cubic-bezier(0.22, 0.61, 0.36, 1)` |

### 新規ページへの variant 割り当て方針

同一ページ内で variant が重複しないよう、セクション数に応じて 1〜6 を分散させる。色は `#FDD000` / `#F08300` / `rgba(240,131,0,0.6)` / `rgba(178,186,230,0.8)` などを交互に使う。variant 4・6 はページ中盤〜後半のアクセントポイントに1箇所ずつ。

## 6. Blob 装飾（BlobDecoration コンポーネント）

### 基本思想
**画面内に収めない。はみ出す。重なる。隠れる。**

Blob が表現するのは**生き生きとした成長感・人間的な温かさ**であり、装飾的なかわいさではない。

### 配置・数の目安

| ページの長さ | Blob の目安数 |
|------------|-------------|
| 4〜6セクション | 2〜3個 |
| 7〜10セクション | 3〜4個 |
| 10セクション以上 | 4〜5個（置かないセクションを意図的に作る） |

- **白背景のみのセクションにはBlobを置かない**のを基本とする（クリーム帯域に集中させる）
- ただし HeroSection 周辺は白でも可（起点として生き生きした印象を出す）
- **同一ページで同じ shape を連続使用しない**
- **同一ページで同じ color を隣接セクションで繰り返さない**
- 配置は「右上・左下・右下・左上」をローテーション

### Blob カラーパレット

| カラー | 用途 |
|--------|------|
| `rgba(253, 232, 166, 0.28)` | クリーム・イエロー |
| `rgba(235, 180, 178, 0.2)` | ピンク |
| `rgba(178, 186, 230, 0.2)` | ラベンダー |
| `rgba(249, 115, 22, 0.25)` | オレンジ（CTA・強調） |

透明度は 0.15〜0.28 の範囲に保つ。

### Blob はセクション境界をまたいで漂う（overflow-hidden 禁止）

Blob を含むコンテナに `overflow-hidden` を付けると Blob が直線でクリップされ不自然に途切れる。

```tsx
{/* NG */}
<div className="relative overflow-hidden">

{/* OK */}
<div className="relative">
```

## 7. 画像・セクションの切り取り方

**画像は大胆にトリミングする。**

- `hero-photo` のように角丸を片側だけ大きくする（`border-radius: 0 0 0 12rem`）
- 画像がセクション境界をはみ出す構成
- `object-cover` で被写体が画面端で切れることを恐れない

## 9. セクション区切り・レイヤリング

### WaveDivider の使い方

**「背景色が変わる境界」にのみ置く。同じ背景色が続く境界には置かない。**

WaveDivider はそれ自体に高さを持つため、不要な場所に置くと大きな余白が生まれる。

| 状況 | WaveDivider |
|------|-------------|
| 白 → クリームに変わる | ✅ 置く |
| クリーム → 白に変わる | ✅ 置く |
| 白 → 白（同色） | ❌ 置かない |
| ArchClip を使うセクションの直前 | ❌ 置かない（ArchClip 自体が境界を担う） |

### WaveDivider の色の決め方

`bgColor` = **手前のセクションの背景色**、`fillColor` = **次のセクションの背景色**

```tsx
{/* クリーム → 白 */}
<WaveDivider bgColor="#fffdf5" fillColor="#ffffff" />

{/* 白 → クリーム */}
<WaveDivider bgColor="#ffffff" fillColor="#fffdf5" />

{/* 波の向きを反転したい場合 */}
<WaveDivider bgColor="#fffdf5" fillColor="#ffffff" flip />
```

### HeroSection と直下セクションの背景を馴染ませる

HeroSection 自体は `bg-transparent`。直下セクションと視覚的に一体化させたい場合は共通の背景色を持つ wrapper div で包む。

```tsx
<div className="relative bg-[#fffdf5]">
    <HeroSection title="..." description="..." activeIndex={N} />
    <SomeIntroSection ... />
</div>
```

### ArchClip を使うセクションのルール

**① `section` 要素の `bg-*` は前セクションの背景色に合わせる**

ArchClip はあくまで「内側コンテンツ」をクリップするだけで、`section` 要素自体には作用しない。アーチより上の三角形は `section` の `bg-*` がそのまま見えるため、前セクションの色と必ず揃えること。

```tsx
{/* NG: section が bg-gray-900 だとアーチ上部も暗くなる */}
<section className="bg-gray-900">

{/* OK: 前セクションが #fffdf5 なら section も同じ色に */}
<section className="bg-[#fffdf5]">
    <ArchClip>…暗い背景…</ArchClip>
</section>
```

**② ArchClip を使うセクションの直前に WaveDivider を置かない**

## 13. NGリスト

- **インラインのグラデーション棒（`span` で `w-[80px] h-[4px] rounded-full` の黄〜オレンジグラデーション）は永久に使用禁止。** ❌
- **`.divider-line`（グラデーション直線）は使用禁止。見出し下・セクション区切りのラインは HandwrittenLine のみ使う。** ❌
- 光沢感・輝き感のあるギラついたグラデーションを使う ❌
- 見出しに `text-balance` を付けない ❌
- 本文に `text-pretty` を付けない ❌
- ページコンテンツで uppercase を多用する ❌
- prefers-reduced-motion 対応を書き忘れる ❌
- 同一ページで Blob を同じ color・同じ shape で連続する ❌
- 画像をセクション内に「きれいに収める」だけにする ❌
- **ふわふわ・かわいい印象のデザインにする（求職者を軽く見た表現になる）** ❌
- **クライアント企業視点（任せられるか・信頼できるか）を忘れる** ❌
- **セクション間の背景色を直線でぶつける（波型・曲線など滑らかな境界を推奨）** ❌
- **Blob を含むコンテナに `overflow-hidden` を付ける（Blob が直線でクリップされる）** ❌
- **WaveDivider を背景色が変わらない境界に置く（不要な余白・隙間が生まれる）** ❌
- **ArchClip を使うセクションの `section` 要素に暗い `bg-*` を付ける（アーチ上部が暗くなる）** ❌
- **ArchClip を使うセクションの直前に WaveDivider を置く（隙間が生まれる）** ❌
- **HeroSection と直下セクションの背景色が異なるのに wrapper でまとめない（境界が直線になる）** ❌
- **HeroSection 直下（ファーストビュー内）の HandwrittenLine に `delayWhenInViewMs` を付けない（HeroSection アニメーションと重なる）** ❌
- **Recruitページのセクション構成・コンポーネント名・データをそのまま他ページにコピーする** ❌
